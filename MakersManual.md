# Resonance — 制作者向けガイド

## Resonance とは

Resonance は、DLsite で購入した音声作品をローカルで管理・再生するための Windows デスクトップアプリです。  
フォルダをスキャンするだけで作品をライブラリに取り込み、タグ・CV・サークルでのフィルタリング、スマートフォルダ、ABループ、スリープタイマーなど、音声作品に特化した機能を備えています。

> **DLsite 以外の作品でも使えます。** RJ番号やサークルURLなどDLsite固有の項目を除き、ライブラリ管理・再生・分岐・トラック整理・LRC歌詞といった機能はすべて利用可能です。BOOTH やその他のプラットフォームで配布している作品にも対応できます。

**音声作品制作者にとって特に注目すべき機能**は以下の通りです。

- **分岐再生・シナリオ分岐** — トラック終了時に選択肢を表示し、マルチエンディングや難易度選択に対応
- **トラック整理** — 実際のフォルダ構成を変えずに、並び順や章立てをカスタマイズ
- **.rsce / .rsdate ファイル** — 分岐設定やメタデータをファイルとして配布し、リスナー側の設定を不要に
- **LRC 歌詞同期表示** — セリフや状況説明のテキストを再生に同期して表示

## このドキュメントについて

このドキュメントでは、あなたの作品が Resonance 上でどのように取り込まれ、表示・再生されるかを解説します。

フォルダの構成やファイル命名を少し工夫するだけで、リスナーの体験を大きく向上させることができます。  
特に **分岐再生** や **シナリオ分岐** に対応した作品は `.rsce` ファイルを、メタデータを事前に設定したい場合は `.rsdate` ファイルを同梱することで、リスナー側の設定が不要になります。  
これらのファイルは作品フォルダに同梱して配布するだけでなく、後から単体で配布してリスナーに手動インポートしてもらうことも可能です。

## 1. フォルダ構成

### 基本ルール

Resonance は **1 フォルダ = 1 作品** として認識します。  
作品フォルダの中にある音声・動画ファイルがトラックとして登録されます。

### 対応フォーマット

| 種類 | 対応形式 |
|---|---|
| 音声 | mp3, wav, flac, ogg, m4a, aac, opus |
| 動画 | mp4, mkv, avi, mov, wmv, webm, flv, m4v, ts, mpg, mpeg, rm, rmvb, 3gp, divx |

動画ファイルはトラックとして登録されますが、再生時はOS既定の外部プレイヤーで開きます。

```
RJ123456_作品タイトル/
├── 01_導入.mp3
├── 02_本編.mp3
├── 03_エンディング.mp3
├── cover.jpg            ← カバー画像（自動検出）
└── lyrics.lrc           ← 歌詞ファイル（任意）
```

### トラックの並び順

トラックは **ファイルパスの昇順** で並びます。  
ファイル名の先頭に連番を付けることで、意図した順序で再生されます。

```
01_プロローグ.mp3
02_本編A.mp3
03_本編B.mp3
04_エピローグ.mp3
```

### 差分ファイル・複数バージョンの構成

同一トラックの差分（別バージョン、追加音声など）がある場合は、**サブフォルダで分ける** ことを推奨します。  
Resonance はファイルパス順でソートするため、サブフォルダ単位でまとまって再生されます。

```
RJ123456/
├── normal/
│   ├── 01.mp3
│   ├── 02.mp3
│   └── 03.mp3
└── ささやき/
    ├── 01.mp3
    ├── 02.mp3
    └── 03.mp3
```

この場合、`normal/01 → 02 → 03 → ささやき/01 → 02 → 03` の順で並びます。  
フォルダ名のアルファベット・50音順でグループ化されるため、意図した順序にしたい場合はフォルダ名の先頭に番号を付けてください。

```
RJ123456/
├── 01_通常版/
│   └── ...
└── 02_ささやき版/
    └── ...
```

### Scan Directory について

リスナーが「Scan Directory」機能を使う場合、指定フォルダの **直下のサブフォルダ** がそれぞれ 1 作品として認識されます。入れ子構造（例: `サークル名/作品名/`）には対応していないため、作品フォルダが直下に並ぶ構成が前提となります。

## 2. トラック整理（カスタム並び順・仮想フォルダ）

Resonance では、実際のフォルダ構成に関係なく、トラックの **並び順** や **グループ分け** を自由にカスタマイズできます。

### カスタム並び順

デフォルトではトラックはファイルパスの昇順で並びますが、「トラック並び順」モーダルでドラッグ＆ドロップにより任意の順序に変更できます。  
たとえば、ファイル名の都合で意図した順序にならない場合や、複数フォルダのトラックを混ぜて再生したい場合に便利です。

### 仮想フォルダ（カスタムグループ）

トラックを任意の名前のグループに分類できます。実際のフォルダ構成を変更する必要はありません。

たとえば、以下のようなフォルダ構成の作品があるとします。

```
RJ123456/
├── track01.mp3
├── track02.mp3
├── track03.mp3
├── track04.mp3
├── track05.mp3
└── track06.mp3
```

これを Resonance 上で以下のように整理できます。

| 仮想フォルダ | トラック |
|---|---|
| 第1章 | track01.mp3, track02.mp3 |
| 第2章 | track03.mp3, track04.mp3 |
| エピローグ | track05.mp3, track06.mp3 |

リスナーの Tracks タブにはこの仮想フォルダ名でグループ化されて表示され、再生順序もこのカスタム順に従います。

### トラック表示名のカスタマイズ

トラックの表示名を、ファイル名とは別にカスタマイズできます。実ファイルのリネームは不要です。

- **設定方法:** トラックを右クリック →「表示名を変更」、またはトラック整理モーダルでダブルクリック
- 設定した表示名は、トラック一覧・キュー・分岐選択肢など、アプリ内のあらゆる場所で使われます
- ファイル名は伏せて、シーン説明や台詞の一節を表示名にする、といった演出が可能です

> **活用例:** ファイル名は `track03.mp3` のままにして、表示名を「廊下での出会い」に変更すると、リスナーには物語的な見出しとして提示できます。

### 配布

カスタム並び順・仮想フォルダ・トラック表示名の情報は、分岐・シナリオ設定と同じ [.rsce ファイル](#7-rsce-ファイル分岐データの配布) に含まれます。  
作品フォルダに `.rsce` を同梱するだけで、リスナー側でもこの整理済みの状態が自動的に再現されます。

> **活用例:** フラットなフォルダ構成で配布しつつ、`.rsce` で章立て・再生順序・表示名を指定することで、リスナーに意図した構成で体験してもらえます。

## 3. カバー画像・スライダー画像

### カバー画像の自動検出

カバー画像は DLsite の作品登録用メイン画像と同じものを想定しています（アスペクト比 4:3、推奨サイズ 560x420）。  
作品フォルダ内に画像ファイルを置くと自動検出されます。以下のファイル名が **優先的に** 選択されます。  
なお、リスナーが作品詳細パネルの「編集」タブからカバー画像を手動で変更することもできます。

| 優先されるファイル名 | 例 |
|---|---|
| `cover` | cover.jpg, cover.png |
| `folder` | folder.jpg |
| `artwork` | artwork.png |
| `front` | front.jpg |
| `thumb` | thumb.png |

上記に該当しない場合は、フォルダ内の最初の画像ファイルが使用されます。

> **リサイズ:** 縦横サイズの合計（幅+高さ）が 1000px 超、またはファイルサイズが 300KB 超の画像は、自動的にサムネイルサイズ（幅 300px）にリサイズされて保存されます。元の画像ファイルは変更されません。

### その他画像（紹介画像・スライダー画像）

作品フォルダ内にある画像ファイルは、カバー画像以外もすべて「その他画像」として認識されます。  
サブフォルダ内の画像も再帰的に検出されるため、フォルダ構成は自由です。  
DLsite の商品ページに掲載している紹介画像を入れておくと、リスナーが詳細パネルやビューアで閲覧できます。

```
RJ123456/
├── cover.jpg
├── 紹介画像/
│   ├── slide01.jpg
│   ├── slide02.jpg
│   └── slide03.jpg
└── 01_本編.mp3
```

- 詳細パネル上部のサムネイルをクリック or マウスホイールで切り替え表示

## 4. LRC 歌詞ファイル

音声ファイルと同名の `.lrc` ファイルを配置すると、再生中にシークバー下に歌詞が同期表示されます。

```
RJ123456/
├── 01_本編.mp3
├── 01_本編.lrc    ← 同名の .lrc ファイル
├── 02_後編.mp3
└── 02_後編.lrc
```

### LRC フォーマット

標準的な LRC タイムタグ形式に対応しています。

```
[00:00.00]（ドアを開ける音）
[00:03.50]おはよう、今日も来てくれたんだね
[00:08.20]さあ、座って座って
```

タイムタグは `[mm:ss.xx]` 形式です。

> **活用例:** 音声作品では、セリフや状況説明のテキストを LRC で同期表示することで、聴き取りにくい部分の補助やシーン解説として活用できます。

## 5. 分岐再生

Resonance はトラック終了時に **複数の選択肢** を表示し、リスナーが次の再生先を選ぶ「分岐再生」に対応しています。  
マルチエンディングや会話分岐のある作品で活用できます。

### リスナー側の体験

1. 分岐ポイントのトラック再生が終了する
2. 画面中央に選択肢パネルが表示される
3. リスナーがクリックまたは数字キーで選択
4. 選んだ先のトラックが再生される

### 分岐に適したフォルダ構成の例

```
RJ123456/
├── 01_導入.mp3
├── 02_分岐前.mp3      ← このファイルの再生終了時に選択肢が表示される
├── 03A_ルートA本編.mp3
├── 03B_ルートB本編.mp3
├── 04A_ルートAエンド.mp3
└── 04B_ルートBエンド.mp3
```

### 設定項目

分岐ポイントには以下の設定が可能です。

| 項目 | 説明 |
|---|---|
| 選択肢ラベル | リスナーに表示されるテキスト（例: 「ルートA: 一緒に帰る」）。未設定の場合は移動先のトラック名がそのまま表示される |
| 移動先トラック | その選択肢を選んだときに再生されるトラック |
| 重み | ランダム選択時の確率配分（デフォルト: 1） |
| タイムアウト | 指定秒数経過で自動遷移（未設定なら選択するまで待機） |

### ランダム遷移

タイムアウトを **0 秒**、タイムアウト時の遷移先を **選択肢からランダム（重み付き）** に設定すると、選択肢を表示せずにランダムで次のトラックへ遷移します。  
リスナーに選択させず、再生のたびに異なる展開を提供したい場合に有効です。各選択肢の重みを調整することで、出現確率を制御できます。

### ランダム選択ボタン

分岐選択肢パネルには、リスナーが能動的に押せる **「ランダム」ボタン** が表示されます。  
押すと、各選択肢の **重み** に応じて 1 つが自動的に選ばれます。

- 制作者側での特別な設定は不要（タイムアウト 0 秒のランダム遷移とは別物）
- リスナーが「自分で決めたくないとき」「運任せにしたいとき」に使える
- 事前分岐選択（後述）にも同じランダムボタンが表示される

### ループと脱出

分岐の移動先に **自分自身のトラック** を含めることで、同じトラックを繰り返し再生するループを作れます。  
選択肢やランダム遷移と組み合わせると、確率や選択で脱出する構成が実現できます。

**例1: リスナーの選択でループを抜ける**

`02_繰り返し.mp3` の分岐設定:

| 選択肢ラベル | 移動先 |
|---|---|
| 「もう一回」 | 02_繰り返し.mp3（自分自身） |
| 「次へ進む」 | 03_次の展開.mp3 |

リスナーが「もう一回」を選ぶ限り同じトラックが再生され、「次へ進む」を選ぶとループを抜けます。

**例2: 確率でループを脱出する**

タイムアウト0秒 + ランダム遷移を使うと、選択肢を表示せずに確率でループするか脱出するかが決まります。

`02_繰り返し.mp3` の分岐設定:

| 選択肢ラベル | 移動先 | 重み |
|---|---|---|
| ループ | 02_繰り返し.mp3（自分自身） | 3 |
| 脱出 | 03_次の展開.mp3 | 1 |

この例では 75% の確率でループし、25% の確率で脱出します。  
シナリオ変数と変数別の重みを組み合わせれば、難易度によってループの脱出確率を変えることもできます。

### 次トラック設定

分岐ではなく、特定トラックの再生後に **固定の1トラック** へジャンプさせたい場合は「次トラック設定」を使います。

| ジャンプ先 | 動作 |
|---|---|
| 任意のトラック | 指定トラックへジャンプ |
| 再生終了 | そのトラックで停止 |
| 元のトラックに戻る（最初から） | 直前に再生していたトラックの先頭から再生 |
| 元のトラックに戻る（位置を復元） | 直前に再生していたトラックの、ジャンプした時点の再生位置から再開 |

これにより、分岐後のルートが合流するポイントや、ボーナストラック後に本編に戻る構成などを表現できます。

### クイックジャンプ

再生中にキーボードの数字キー（<kbd>1</kbd>〜<kbd>9</kbd>）で特定のトラックへ即座にジャンプできるショートカットです。  
トラック終了を待たずにいつでも発動できるため、「好きなシーンにすぐ飛べる」体験を提供できます。

2 種類のクイックジャンプがあります。

| 種類 | 動作 |
|---|---|
| 全トラック共通 | 作品内のどのトラック再生中でも有効 |
| トラック別 | 特定のトラック再生中だけ有効（複数の発動元を指定可） |

トラック別のクイックジャンプには、さらに **有効範囲（開始位置・終了位置）** を設定できます。  
たとえば「2:00〜10:00 の間だけ有効」のように指定すると、そのトラックの該当区間を再生中のときだけ数字キーが反応します。  
作品内の特定のセリフや場面にぴったり合わせてジャンプを発動させたい場合に便利です。

**活用例:**

- チャプター選択 — 全トラック共通で <kbd>1</kbd>=プロローグ、<kbd>2</kbd>=本編、<kbd>3</kbd>=エピローグ のように設定し、好きなシーンへ直接ジャンプ
- 特定シーンからの分岐 — あるトラックの再生中だけ <kbd>1</kbd>=選択肢A、<kbd>2</kbd>=選択肢B を有効にし、リスナーが好きなタイミングで選択

> **分岐選択肢との違い:** 分岐はトラック終了時に自動で選択肢が表示されますが、クイックジャンプはリスナーが能動的に数字キーを押す必要があります。作品内でキー操作の案内をしておくと効果的です。

### 「元のトラックに戻る」とクイックジャンプの組み合わせ

次トラック設定の「元のトラックに戻る」とクイックジャンプを組み合わせると、**本編の途中で寄り道トラックを聴いてから元の位置に戻る** 構成が作れます。

たとえば、本編トラックの再生中にクイックジャンプで「おまけトラック」へ飛べるようにし、そのおまけトラックの次トラック設定を「元のトラックに戻る（位置を復元）」にしておきます。

| 設定 | 内容 |
|---|---|
| 本編トラック | クイックジャンプ <kbd>1</kbd> → おまけトラック |
| おまけトラック | 次トラック設定 → 元のトラックに戻る（位置を復元） |

リスナーの体験:
1. 本編を再生中に <kbd>1</kbd> を押す → おまけトラックにジャンプ
2. おまけトラックの再生が終了 → 自動的に本編の中断した位置から再開

「位置を復元」を使うことで、本編の流れを完全に維持したまま寄り道できます。「最初から」を選べば、戻った際にトラックの頭から再生されます。

用語集・キャラ紹介・サウンドテスト的なコンテンツを本編の流れを止めずに提供したい場合に有効です。

### 「分岐を指定して再生」（シナリオ無し作品向け）

シナリオ設定を行っていない作品でも、分岐ポイントが 1 つでもあれば「分岐を指定して再生」ボタンが表示されます。  
リスナーは **開始トラックと、各分岐での選択肢を事前に指定** してから再生を開始できます。

- シナリオ機能を使わずに、シンプルに「マルチエンディングの行先を先に決めて流したい」場合に便利
- 制作者側での追加設定は不要（分岐ポイントを設定しているだけで自動的に有効）

## 6. シナリオ分岐

「シナリオ分岐」は、作品ごとに **シナリオ変数** を定義し、分岐の選択肢に出現条件を設定する機能です。  
**複数の変数を独立したフラグとして扱える** ため、難易度・ルート・好感度などを組み合わせた複雑な分岐構成が可能になっています。

### 構成要素

| 要素 | 説明 |
|---|---|
| シナリオ変数 | 作品に **複数定義できる** 名前付き変数（例: 「難易度」「ルート」「好感度」） |
| 値リスト | 各変数が取りうる値の一覧（例: 難易度 = やさしい / ふつう / ハード） |
| シナリオ開始ボタン | リスナーが最初に選ぶボタン。1 ボタンで複数変数の値をまとめてセット可能 |
| 条件付き選択肢 | 各分岐の選択肢に **複数変数を AND/OR で組み合わせた出現条件** を設定可能 |
| 選択時の変数セット | 選択肢が選ばれたときに **複数の変数を同時に変更** 可能 |

> **v0.9.2 以前との違い:** 旧バージョンは「変数 1 つ・値 1 つマッチ」でしたが、現在は複数変数を AND/OR 条件で組み合わせられます。既存のシナリオ設定は自動的に新形式へ移行されます。

### シナリオ開始ボタン — 最初に提示する選択肢

シナリオ開始時にリスナーへ提示するボタンを **自由に定義** できます。1 つのボタンで **複数の変数を同時にセット** することも可能です。

例: 「やさしい・甘えん坊ルート」というボタンを 1 つ定義し、押すと「難易度 = やさしい」「ルート = 甘えん坊」を同時にセットする、といった構成が作れます。

設定箇所: 作品詳細パネル → シナリオタブ → 「開始ボタン」一覧

各ボタンには以下を設定します。

| 項目 | 説明 |
|---|---|
| ボタンラベル | リスナーに表示される文言 |
| セットする変数 | このボタンが押されたときに値をセットする変数（複数可） |
| 開始トラック | 押したときに再生を開始するトラック（未指定なら作品の最初） |

### 通常モード — 最初にルートを選ぶ

シナリオ開始時にルート選択画面（開始ボタン）が表示されるパターンです。

```
RJ123456_耳かき体験/
├── 01_来店.mp3
├── 02_施術開始.mp3
├── 03_やさしい.mp3          ← 「やさしい」選択時のみ再生
├── 03_ふつう.mp3            ← 「ふつう」選択時のみ再生
├── 03_ハード.mp3            ← 「ハード」選択時のみ再生
├── 04_仕上げ.mp3
├── 05_やさしい_お見送り.mp3  ← 「やさしい」選択時のみ再生
├── 05_ふつう_お見送り.mp3    ← 「ふつう」「ハード」選択時のみ再生
└── 06_エンディング.mp3
```

条件に複数の値を指定することもできます。上の例では `05_ふつう_お見送り.mp3` に「ふつう」と「ハード」の両方を条件として設定すると、どちらのルートでもこのトラックが再生されます。

リスナーの体験:
1. 「シナリオを始める」ボタンを押す
2. 「やさしい」「ふつう」「ハード」のボタンが表示される
3. 選んだルートに応じて、分岐の選択肢が自動的にフィルタリングされる

### 選択肢ごとのシナリオ変数設定

分岐の各選択肢には、以下のシナリオ関連設定ができます。

| 設定 | 説明 |
|---|---|
| **出現条件（AND/OR ツリー）** | 複数の変数条件を AND/OR で組み合わせた論理ツリーで、表示するかを判定 |
| **選択時に変数を設定** | この選択肢が選ばれたときに **複数の変数を同時にセット** 可能 |
| **変数別の重み** | 特定の変数値ごとにランダム選択の重みを個別に設定できる |

「出現条件」は選択肢のフィルタリング、「選択時に変数を設定」は変数値の変更に使います。  
この 2 つを組み合わせることで、以下のような構成が可能です。

- 最初の分岐で複数変数を **同時セット** し、以降の分岐で **AND/OR 条件で参照** する
- 途中の分岐で変数を **別の値に上書き** して、後半の展開を変える
- 条件なし（常に表示）の選択肢と、特定の条件下でのみ出現する選択肢を **混在** させる
- ランダム遷移時に、難易度ごとに異なる確率で選択肢が選ばれるようにする（変数別の重み）

#### AND/OR 条件ツリーの例

| 条件式 | 意味 |
|---|---|
| `難易度 = ハード` | 難易度がハードのときのみ表示 |
| `難易度 = ハード AND ルート = 甘えん坊` | ハード かつ 甘えん坊ルートのときのみ表示 |
| `(難易度 = ハード) OR (好感度 = 高い)` | ハード または 好感度が高いとき表示 |
| `ルート = 甘えん坊 AND (好感度 = 高い OR 好感度 = 普通)` | 甘えん坊ルート かつ 好感度が高い・普通のとき表示 |

複数変数を組み合わせることで、「ハードルート × 高好感度限定の隠しシーン」のような細かい分岐演出が可能になります。

> **変数別の重みの活用例:** タイムアウト0秒+ランダム遷移の分岐で、やさしいでは「成功」の重みを高く、ハードでは「失敗」の重みを高く設定することで、難易度に応じてランダムイベントの成功率を変える、といった構成が可能です。

### ハイドモード — 途中で難易度が決まる

難易度選択を表示せず、物語途中の分岐選択肢で難易度が決まるパターンです。

```
RJ123456_ハイド/
├── 01_プロローグ.mp3
├── 02_分岐点.mp3            ← このファイルの再生終了時に難易度を決定
├── 03_やさしい.mp3
├── 03_ハード.mp3
├── 04_合流.mp3              ← 難易度共通
├── 05_分岐点2.mp3           ← 難易度に応じて異なる選択肢が出る
├── 06_やさしい結末.mp3
└── 06_ハード結末.mp3
```

`02_分岐点.mp3` の分岐設定例:

| 選択肢ラベル | 移動先 | 選択時に変数を設定 |
|---|---|---|
| 「やさしくして」 | 03_やさしい.mp3 | → やさしい |
| 「もっと強く」 | 03_ハード.mp3 | → ハード |

`05_分岐点2.mp3` の分岐設定例:

| 選択肢ラベル | 移動先 | 出現条件 |
|---|---|---|
| 「おだやかな結末へ」 | 06_やさしい結末.mp3 | やさしい |
| 「激しい結末へ」 | 06_ハード結末.mp3 | ハード |

リスナーの体験:
1. 「シナリオを始める」ボタンで 01 から再生開始
2. 02 終了時に選択肢が表示され、選択により難易度が内部的にセットされる
3. 以降の分岐ポイントでは、セットされた難易度に該当する選択肢だけが表示される

> **ポイント:** `05_分岐点2.mp3` では出現条件を満たす選択肢が 1 件だけになるため、選択 UI は表示されず自動的に遷移します。リスナーから見ると、難易度に応じて自然にストーリーが分岐しているように感じられます。

### 難易度によって選択肢が変わる例

出現条件を活用すると、同じ分岐ポイントでも難易度によって **表示される選択肢の数や内容が変わる** 構成が作れます。

たとえば、中盤の分岐でリスナーに行動を選ばせるシーンを考えます。

`04_中盤の分岐.mp3` の分岐設定例:

| 選択肢ラベル | 移動先 | 出現条件 |
|---|---|---|
| 「休憩する」 | 05_休憩.mp3 | （条件なし = 常に表示） |
| 「そのまま続ける」 | 05_続行.mp3 | （条件なし = 常に表示） |
| 「もっと激しく」 | 05_激しい.mp3 | ハード |

この場合のリスナーの体験:

- **やさしい・ふつう** → 「休憩する」「そのまま続ける」の 2 択が表示される
- **ハード** → 「休憩する」「そのまま続ける」「もっと激しく」の 3 択が表示される

条件なしの選択肢と条件付きの選択肢を混在させることで、特定の難易度でだけ追加の選択肢が出現する構成を実現できます。

### 事前分岐選択（先に全部選んでおく）

シナリオ開始時に、**作品中のすべての分岐を先にまとめて選択** してから再生を開始できる機能です。  
リスナーが「途中で選択ダイアログに止められたくない、流して聴きたい」場合に有効です。

- シナリオ開始時、「先に分岐を決めておく」というトグルスイッチが表示されます
- 開始ボタンを押した直後に、作品中の全分岐ポイントの選択肢を順に提示
- 全部選び終わると、最初から最後まで自動的にその選択通りに再生される
- 各分岐パネルにはランダム選択ボタンも表示されるので、一部だけ運任せにすることも可能

> **制作者側の追加設定は不要:** シナリオ設定がされていれば自動的に有効になります。「分岐を指定して再生」（シナリオ無し作品向け）と似た UI ですが、こちらはシナリオ変数とリンクするため、変数の出現条件によって表示される分岐がフィルタリングされます。

### 分岐の評価ロジック

分岐ポイントに到達したとき、現在のシナリオ変数の値が **AND/OR 条件ツリー** で評価され、選択肢が絞り込まれます。

| マッチ件数 | 動作 |
|---|---|
| 1 件 | 自動選択して即遷移（選択 UI を表示しない） |
| 2 件以上 | 選択肢パネルを表示 |
| 0 件 | 分岐を無視して通常の次トラック処理へ |

条件が 1 件だけの場合は **選択肢を表示せず自動遷移** するため、リスナーから見ると変数の状態に応じて自然にストーリーが分かれる体験になります。

## 7. .rsce ファイル（分岐データの配布）

`.rsce`（Resonance Scenario Export）は、分岐・シナリオ設定を JSON 形式で保存したファイルです。  
**作品フォルダに `.rsce` ファイルを同梱して配布すると、リスナーがインポート時に分岐設定が自動的に適用されます。**

### 制作者のワークフロー

1. Resonance で作品を取り込み、分岐・シナリオを設定する
2. 作品詳細パネルの「分岐」タブから **エクスポート** する
3. 出力された `.rsce` ファイルを作品フォルダに入れて配布する

### リスナー側の体験

- 作品フォルダをインポートすると `.rsce` ファイルが **自動検出・自動適用** される
- リスナー側での分岐設定は一切不要
- 複数の `.rsce` ファイルがある場合は、最も更新日時が新しいものが使用される
- すでにライブラリに登録済みの作品でも、作品詳細パネルの「分岐」タブから **手動でインポート** できる（`.rsce` ファイルを後から配布する場合にも対応可能）

### .rsce に含まれるデータ

| データ | 内容 |
|---|---|
| 分岐ポイント | 選択肢ラベル・移動先・重み・タイムアウト・出現条件（AND/OR ツリー）・選択時の変数セット |
| シナリオ設定 | 複数の変数定義（変数名・値リスト）・開始ボタン定義（複数）・ハイドモード |
| 次トラック設定 | トラック単位のジャンプ先 |
| クイックジャンプ | 数字キーでのジャンプ先（全トラック共通／トラック別の有効範囲付き） |
| カスタム並び順 | トラックの再生順序 |
| 仮想フォルダ | トラックのグループ名 |
| トラック表示名 | ファイル名と別に設定したトラックのカスタム表示名 |

### トラックのマッチング

`.rsce` ファイルはトラックを **相対パス** で参照します。  
インポート先の作品フォルダ内に同じ相対パスのファイルがあれば自動的にマッチします。  
ファイルが見つからない設定はスキップされるため、部分的な差分構成でも安全にインポートできます。

### 配布例

```
RJ123456_分岐音声作品/
├── cover.jpg
├── RJ123456.rsce          ← これを同梱するだけ
├── 01_プロローグ.mp3
├── 02_分岐点.mp3
├── 03A_ルートA.mp3
├── 03B_ルートB.mp3
└── 04_エンディング.mp3
```

> `.rsce` ファイルは JSON テキストなのでファイルサイズは数 KB 程度です。配布物のサイズにはほぼ影響しません。

## 8. .rsdate ファイル（メタデータの配布）

`.rsdate`（Resonance Data Export）は、作品のメタデータを JSON 形式で保存したファイルです。  
`.rsce` が分岐・シナリオ設定を扱うのに対し、`.rsdate` は作品情報そのものを扱います。  
**作品フォルダに `.rsdate` ファイルを同梱して配布すると、リスナーがインポート時にメタデータが自動的に適用されます。**

### .rsdate に含まれるデータ

| データ | 内容 |
|---|---|
| タイトル | 作品タイトル |
| RJ番号 | DLsite の作品番号 |
| サークル | サークル名・DLsite URL |
| シリーズ | シリーズ名・DLsite URL |
| CV | 出演声優の一覧 |
| タグ | ジャンルタグの一覧 |
| リリース日 | 販売日 |
| メモ | 自由テキストのメモ |

### 制作者のワークフロー

1. Resonance で作品を取り込み、メタデータを設定する
2. 作品詳細パネルの「編集」タブから **.rsdate Export** する
3. 出力された `.rsdate` ファイルを作品フォルダに入れて配布する

### リスナー側の体験

- 作品フォルダをインポートすると `.rsdate` ファイルが **自動検出・自動適用** される
- リスナー側でのメタデータ入力は不要
- 複数の `.rsdate` ファイルがある場合は、最も更新日時が新しいものが使用される
- すでにライブラリに登録済みの作品でも、「編集」タブから **手動でインポート** できる（適用する項目を選択可能）

### .rsce との併用

`.rsdate` と `.rsce` は独立したファイルなので、両方を同梱することでメタデータと分岐設定の両方を配布できます。

```
RJ123456_分岐音声作品/
├── cover.jpg
├── RJ123456.rsdate        ← メタデータ
├── RJ123456.rsce          ← 分岐・シナリオ設定
├── 01_プロローグ.mp3
├── 02_分岐点.mp3
├── 03A_ルートA.mp3
├── 03B_ルートB.mp3
└── 04_エンディング.mp3
```

> どちらも JSON テキストで数 KB 程度のため、配布物のサイズにはほぼ影響しません。
